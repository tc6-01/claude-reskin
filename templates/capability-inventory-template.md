# Capability Inventory

> This file is the **single source of truth** for this reskin. All subsequent changes MUST explicitly reference capability IDs from this file.

## Field Descriptions

| Field | Meaning |
|-------|---------|
| Capability ID | Unique identifier, format `CAP-XXX` |
| Capability Name | Business capability name (NOT filename, NOT module name) |
| Current Definition Owner | Who defines the rules/structure/semantics |
| Current Execution Owner | Who actually performs the operations |
| Dependents | Which other capabilities depend on this one (list capability IDs) |
| Replacement Risk | High/Medium/Low |
